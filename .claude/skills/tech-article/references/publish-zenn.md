# 公開準備: Zenn frontmatter と状態コメントの除去

記事の本文・構造が完成したあと、Zenn に公開するための仕上げを行うフェーズ。ここで実施するのは **frontmatter の確定** と **状態管理用 HTML コメントの除去** の 2 つ。

## Zenn の frontmatter

Zenn 記事は先頭に YAML frontmatter を置く。このリポジトリでは既存記事の雛形がすでに `npx zenn new:article` で生成されており、以下のキーを持つ。

```yaml
---
title: "記事タイトル"
emoji: "🎃"
type: "tech" # tech: 技術記事 / idea: アイデア
topics: []
published: false
---
```

各キーの役割と確定時の判断基準は次のとおり。

**title**: 記事タイトル。ピラミッドの主ポイント（= SCQA の答え）を短く言い換えた文にするのが原則。白紙の主張（「N つの〇〇」だけで中身なし）にしない。ユーザーが書き下した主ポイントをタイトル候補として提示し、調整する。

**emoji**: 記事アイコン用の絵文字 1 文字。Zenn の仕様として 1 文字に限る。内容に沿った絵文字をユーザーに選ばせる。デフォルト（🎃 など生成時の仮値）のままにしない。

**type**: 技術記事は `tech`、ポエムや考察は `idea`。このスキルは技術記事前提なので基本 `tech`。ユーザーが明示的にアイデア記事と指定した場合のみ `idea` にする。

**topics**: タグの配列。Zenn の慣例に沿って小文字で記述する（`["react", "typescript", "nextjs"]` のような形）。記事の主題に含まれる技術・ツール・言語を 3〜5 個程度選ぶ。付けすぎると焦点がぼやけるので厳選する。

**published**: `true` で公開、`false` で下書き。デフォルトは下書きの `false`。ユーザーから明示的に「公開する」と指示があって初めて `true` に切り替える。自動で切り替えない。

**publication_name**: 所属 Publication がある場合のオプション。リポジトリ内の既存記事を参照して、そのユーザーが利用している Publication 名を確認する。必要ない記事では省略する。

## 状態管理用 HTML コメントの除去

企画・構造・導入など各対話フェーズで、記事ファイルの末尾に HTML コメントとして状態を保持している（`DESIGN.md` の「状態管理」参照）。成果物と履歴からなるこのコメントブロックは **公開記事には不要** なので、公開準備フェーズで完全に除去する。

除去は公開準備の最終ステップとして行う。具体的な流れ:

1. `published: true` への切り替えをユーザーが承認したタイミングでコメントブロックを削除
2. 削除後、frontmatter と本文だけの状態になっていることを目視確認
3. `npx zenn preview` をユーザーに勧めて最終確認を依頼

コメントブロックを削除する前に **必ず別フェーズへの往復が完了していることを確認する**。往復中（たとえば検証サブエージェントの指摘を反映中、推敲の途中など）に削除してしまうと、次回以降の作業で状態を復元できなくなる。`published: true` と削除を同時に行うのが原則。

## 公開準備フェーズの対話フロー

オーケストレーターはこのフェーズで以下の順に確認する。

最初に、**記事が本当に公開準備段階に来ているか** を確認する。検証・推敲サブエージェントを一度も実行していないなら、実行を推奨する（押し付けはしない）。ユーザーが「もう公開する」と判断したなら先へ進む。

次に、frontmatter の各キーを順に確定する。title はピラミッドの主ポイントから提案する。emoji は候補を複数提示して選ばせる。topics は記事本文から拾った技術・ツール名を候補として提示する。publication_name は既存記事を見て自動で埋めるか、該当しないなら省略する。

最後に、**公開の最終承認** を明示的に得てから `published: true` に切り替え、状態コメントを削除する。その後 `npx zenn preview` によるローカル確認を案内する。
